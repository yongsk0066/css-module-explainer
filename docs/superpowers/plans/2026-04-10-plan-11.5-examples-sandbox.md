# Plan 11.5 — `examples/` Dogfood Sandbox (Phase 11.5)

**Goal:** Ship an `examples/` directory with a Vite+ root and per-scenario sub-packages so the extension can be manually dogfooded against every pattern the parser claims to support. This is the "real app" sandbox the spec calls out in §8.6 and G8 but that never made it into an implementation plan.

**Architecture:** Single `examples/` folder under the repo root containing (1) a shared Vite+ `run.tasks` config for orchestrating scenarios, (2) a private workspace `package.json` that does not leak into the published extension, (3) nine scenario sub-packages (one per cx() pattern family), and (4) a `README.md` describing how to use the sandbox during provider / parser development. Each scenario has a `pnpm vite` fallback so Vite+ alpha regressions never block dogfooding. None of the scenarios are automated test targets — Tier 1 / Tier 2 / Tier 3 cover that.

**Tech stack:** `vite ^7.x` (or whatever ships with Vite+ 0.1.x), `viteplus ^0.1.x`, `react ^19` (for the JSX scenarios), TypeScript inheriting from the repo root tsconfig. `viteplus` is **alpha**; the fallback path uses plain `vite` directly.

## Non-goals

- **Not** an automated test suite. vitest does not run inside `examples/`.
- **Not** part of the extension bundle. `.vscodeignore` excludes `examples/` from the VSIX.
- **Not** a monorepo workspace that the root `pnpm install` traverses into. Each scenario is independently installable via `pnpm -C examples/scenarios/01-basic install` so marketplace-CI time stays fast.
- **Not** a substitute for the Tier 3 `test/e2e/workspace/` fixture (Plan 10.5). Tier 3 is frozen; examples evolve.

```
    └── 09-large-component/           # 100+ cx() calls for perf smoke
```

**Acceptance:**
- `cd examples && pnpm install` installs Vite+ + vite + react for each scenario.
- `cd examples/scenarios/01-basic && pnpm vite` launches a dev server with the extension attached in the debug host.
- `cd examples && pnpm viteplus run all` (Vite+ path) launches every scenario.
- `examples/` is listed in `.vscodeignore` so `vsce package` does not include it.
- `examples/` is **excluded from oxlint + oxfmt** — they follow their own style (React conventions) and would otherwise trip the server lint profile.
- All 9 scenarios have a reference `.module.scss|css` so the extension has something to resolve against.
- **No scenario is required to compile cleanly under our `tsconfig`** — each has its own tsconfig with `noEmit: true` + `skipLibCheck: true` for iteration speed.

**File count:** ~35 files. ~800 LOC total (spec §3.12 estimate was 900).

## Risks

- **Vite+ alpha**: spec R8. Mitigated by the `pnpm vite` fallback per scenario — each `scenarios/<n>/package.json` has a direct `"dev": "vite"` script so Vite+ can regress without blocking dogfooding.
- **Bundling leakage**: `.vscodeignore` MUST list `examples/` or `vsce package` will balloon the VSIX.
- **Lint leakage**: oxlint/oxfmt currently scan the whole tree. `examples/` must be excluded or the React JSX + CSS syntax will false-positive.
- **Scenario drift**: stub READMEs for 03-09 lock the numbering so a Plan-12 PR doesn't accidentally rename things.
